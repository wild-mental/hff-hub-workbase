## 한 줄 요약

**정확성 동력 엔진** — 모든 비교 결과의 출처와 산식을 의심하고 추적하는 30~40대 남성 분석가. **데이터 신뢰 자산의 최종 검증자.**

## 프로필

| 항목 | 값 |
|---|---|
| 나이 | 30대 후반 ~ 40대 |
| 직업 | 분석가/엔지니어/연구원 |
| 가구 | 1~3인 가구 |
| 디지털 친숙도 | 매우 높음 (개발자급) |
| 세그먼트 | Q1-B (인지 高 / 경험 有, 정확성 동기) |
| 1인 LTV (추정) | 약 41.8만 원 |

## 핵심 Pain

| ID | Pain |
|---|---|
| **CORE-5** | 데이터 신뢰 부재 — 비교 앱 자체를 못 믿음 |
| **CORE-4** | 결정 기준 부재 — 산식·기준이 불투명 |
| **EXT-3** | 부작용·약물 상호작용 우려 (자체 검증 욕구) |

## Goal

> "어떤 비교 결과를 봤을 때, 그 숫자의 출처와 산식을 추적할 수 있고, 잘못된 정보가 있으면 신고할 수 있어야 한다."

## 감정 곡선

```
  ① 인지 ── ② 탐색 ─★── ③ 결정 ── ④ 복용 ── ⑤ 재구매
  중립        의심·검증     확신       확인 반복   자발 신고
              ★ 산식 추적                          (F4 신고 루프)
```

## JTBD Job Story

> so I can "데이터를 못 믿는 죄책감" 없이 사용할 수 있다.

## 카피 톤 가이드

- 산식·출처 우선: "1일 단가 = 표시량(mg) × 가격 ÷ (정수 × 정당 함량)"
- "검증된", "추정"의 차이 명확화: ✅/⚠️/🔴 마커
- 메타데이터 노출 (마지막 업데이트 시각, 출처 URL)
- 기술 어휘 OK: "RAG", "Vector Search" 같은 단어 가능

## 결론·시사점

- E2는 매출은 작지만 **데이터 신뢰 자산의 최종 검증자**. 1건의 신고 = KSF-1·4 강화.
- 본 사업의 [`ksf.md`](../concepts/ksf.md) KSF-1 (데이터 신뢰성)·KSF-4 (전문가 큐레이션)의 운영 검수자 역할.
- E2가 떠난다 = 데이터 품질이 낮다는 신호 → 모니터링 KPI: E2 세그먼트 D30 리텐션.
- F4 신고 시스템의 응답 속도(SLA 24h)가 E2 만족도 = 사업 신뢰도 = 광고 단가의 상한.
