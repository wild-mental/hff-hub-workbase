## 한 줄 정의

**모든 비교/검증 결과의 출처·산식·갱신 시각·검증 수준을 노출하고, 사용자 신고 1건 → 24시간 내 응답으로 잇는 신뢰 자산 시스템.**

## 해결 Pain

| Pain ID | 페르소나 | 동작 |
|---|---|---|
| **CORE-4** | C2, N1, E2 | 결정 기준 부재 → 출처/산식 노출 |
| **CORE-5** | C1, E2 | 비교 앱 자체를 못 믿음 → 추적 가능 데이터 |
| **CJM-4** | C2, E1 | 결정 직전 마지막 회의 → 식약처 #####로 안심 |
| **EXT-3** | C2, E2 | 부작용/상호작용 우려 → 1차 표시 + 약사 검수(Phase 2) |

## 핵심 AC

| ID | 기준 |
|---|---|
| F4-AC-1 | 모든 숫자에 출처 링크(식약처 OPEN-API URL 또는 논문 DOI) |
| F4-AC-2 | 모든 비교에 산식(공식) 1줄 노출 |
| F4-AC-3 | 검증 수준 마커 ✅ 공개 조사 / ⚠️ 추정 / 🔴 미검증 의무 표시 |
| F4-AC-4 | 마지막 갱신 시각 KST + 다음 갱신 예정 시각 |
| F4-AC-5 | 1-Tap "이 데이터에 오류 신고" → 폼 1개 (URL + 코멘트) |
| F4-AC-6 | 신고 → 24h 이내 인간 검수 응답 (이메일 + 앱 알림) |
| F4-AC-7 | 신고 처리 결과는 공개 페이지에 누적 (투명성 강화) |
| F4-AC-8 | 매월 1회 "데이터 품질 리포트" 공개 (총 신고 / 처리 / 반영) |

## NFR / 임계치

| 항목 | 임계치 |
|---|---|
| 출처 링크 깨짐 비율 | ≤ 1% (월간 자동 모니터링) |
| 신고 응답 SLA | 24시간 |
| 데이터 갱신 주기 | 가격 24h, 식약처 정보 7d |
| 검증 수준 마커 누락 | 0% (Lint 자동 검출) |
| 인간 검수 처리율 | 월 100% (1인 운영 시 주 평균 < 20건 가정) |

## 외부·내부 의존성

### 외부

- **식약처 OPEN-API** — 권장 복용량, 인증 번호
- **PubMed Open Access** — 논문 DOI (Phase 2)
- **이메일 발송 (Resend / SendGrid)** — 신고 응답
- **약사 자문단** — 신고 검수 (Phase 2: 1인 → Phase 3: 5인)

### 내부

- **F1·F2·F3** 모두 본 시스템에 의존 — 출처/산식/마커가 표준 컴포넌트로 박힘
- **AGENTS.md Lint 워크플로** — 위키 본문에서도 검증 수준 마커 누락 검출

## 리스크

| ID | 리스크 | 완화 |
|---|---|---|
| R1 | 신고 폭주 → 1인 운영 한계 | Phase 1 = 신고 100건/월 가정, 자동 분류 우선 |

## 관련 페르소나

- **★★ E1 나경아** — "식약처 인증" 1줄 신호등
- **★ C1 한정훈** — 산식 신뢰
- **★ N1 조미라** — 전환 합리화 근거

## 결론·시사점

- F4는 본 사업의 **시간 누적 자산** 그 자체. 신고 1건 = 미래 모방 방어선 1건.
- F4 신고 SLA(24h) 가 깨지는 순간 KSF-1·4 동시 붕괴 → MVP에서 신고 응답을 자동화하지 말고 사람이 직접 답하라.
- F4 결과 페이지(데이터 품질 리포트)는 본 사업의 **마케팅 자산** 으로도 활용 — 매월 SEO/PR 콘텐츠 1건.
- F4는 단독으론 가치가 없고 F1·F2·F3 모두에 박혀야 의미를 가짐 — "출처가 따라다니는 비교"가 본 브랜드의 시각적 인장이다.
