## 한 줄 정의

**채널·용량·할인 조건이 제각각인 영양제 가격을 "1일 권장 복용량 기준 원화 단가"로 자동 환산해 1초 안에 비교 가능한 단위로 정규화하는 엔진.**

## 해결 Pain

| Pain ID | 페르소나 | 동작 |
|---|---|---|
| **CORE-1** | C1, E2 | 30분 엑셀 비교 → 1초 자동 환산 |
| **CORE-3** | C2, A2, E1 | 마케팅 단위(고함량·프리미엄) → 표준 단위(mg/일) |
| **CJM-2** | C1, C2 | 탐색 단계 정보 과잉 → 단일 숫자 |
| **CJM-5** | C1, N1 | 재구매 시 가격 변동 자동 감지 |

## 핵심 AC (수용 기준)

| ID | 기준 |
|---|---|
| F1-AC-1 | 입력: 제품 URL 또는 제품명 검색 → 5초 이내 결과 |
| F1-AC-2 | 환산식: `1일 단가 = (가격 ÷ 정수) × (1일 권장량 ÷ 정당 함량)` |
| F1-AC-3 | 환산 오차 ±5% 이내 (vs 사람 검수 기준) |
| F1-AC-4 | 동일 성분 다채널 ≥ 3개(쿠팡·네이버·iHerb) 동시 노출 |
| F1-AC-5 | 결과 화면에 산식과 출처(식약처 권장량) 항상 노출 |
| F1-AC-6 | 단순 모드(E1용): 큰 글씨 1줄 결론 + 신호등 색 |
| F1-AC-7 | 가격 변동 알림 옵션 (재구매·CJM-5 대응) |

## NFR / 임계치

| 항목 | 임계치 |
|---|---|
| TTC (Time to Calculation) | ≤ 5초 (P95) |
| 데이터 신선도 | ≤ 24h (가격), ≤ 72h (재고) |
| 정확도 | ±5% (정량 검수 표본 N=200) |
| 가용성 | 99.5% (1인 운영 현실 고려) |
| 성분 사전 커버리지 | 식약처 등록 6,000+ 성분 중 Top 200 |

## 외부·내부 의존성

### 외부

- **식약처 OPEN-API** — 권장 복용량, 등록 성분 정보
- **쿠팡 파트너스 API** — 가격, 정수, 재고
- **iHerb Affiliate API** — 해외 직구 가격
- **네이버 쇼핑 API** — 가격 비교 데이터
- **Gemini Flash (LLM)** — 단위 표기 정규화 (예: "30정 1,500mg" → "정당 50mg")

### 내부

- **F4 Data Trust System** — 모든 환산 결과의 출처/산식 노출
- **F2 Anti-BS Dashboard** — 동일 화면에서 광고 진위 검증 동시 표시

## 리스크

| ID | 리스크 | 완화 |
|---|---|---|
| R1 | 채널 API 정책 변경 (쿠팡파트너스 종료/제한) | 다채널 분산 + 크롤링 백업 (법적 검토) |
| R2 | LLM 환산 오류 (단위 잘못 인식) | 룰 기반 정규식 1차 + LLM 2차 + 사람 샘플 검수 |
| R3 | 식약처 권장량 표기와 실제 제품 표기 차이 | 권장량 미일치 시 ⚠️ 마커 + 사용자 신고 (F4) |
| R4 | 동일 성분이지만 흡수율 차이 (예: 마그네슘 옥사이드 vs 시트레이트) | Phase 2 — "성분 형태별" 보조 비교 모드 |

## 관련 페르소나

- **★★★ C1 한정훈** — 메인 사용자 (가성비 최적화)
- **★★ C2 박소연** — 적정 가격 기준
- **★★ E1 나경아** — 단순 모드
- **★ E2 김도현** — 산식 검증
- **★ N1 조미라** — 브랜드 전환 트리거 (Phase 4)

자세한 페르소나 매핑은 [`../personas/`](../personas/).

## 결론·시사점

- F1은 본 사업의 **North Star Metric (TTC)** 의 직접 결정자 — F1 품질 = 사업 가치의 50%+.
- F1 단독으론 부족 — F2·F4와 동일 화면에 묶여야 [`../concepts/aos-dos-framework.md`](../concepts/aos-dos-framework.md) Top 5 Pain이 실제로 풀린다.
- MVP 0~6주차 최우선 빌드 항목.
