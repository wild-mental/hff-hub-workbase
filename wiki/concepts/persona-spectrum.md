## 정의

본 프로젝트의 6개 페르소나(C1, C2, A2, E1, E2, N1)를 **인지 수준 × 구매 경험 × 핵심 동기**의 3축으로 한 화면에 펼친 맵.

## 스펙트럼

```
인지 高 │  E2 (데이터 검증)        C1 (가성비 최적화)
        │  Q1-B 정확성              Q1-A 효율성
        │
        │
인지 중 │
        │
        │
인지 低 │  N1 (브랜드 충성)        C2 (건강 계기)
        │  Q3 정서 안정             Q4-A 안전
        │  E1 (디지털 약자)        A2 (트렌드)
        │  Q4-B 가족 보호            Q4-C 자기실현
        └────────────────────────────────────────────
            구매 무경험            구매 유경험
```

## 동기 분류

| 동기축 | 페르소나 | 핵심 한 줄 |
|---|---|---|
| **효율성/최적화** | C1 | "내 시간과 돈, 어떻게 하면 가장 효율적으로 쓸까?" |
| **안전성/책임감** | C2 | "어떻게 하면 내 가족과 나의 건강을 안전하게 챙길까?" |
| **자기실현/탐색** | A2 | "어떻게 하면 더 나은 내가 될 수 있을까?" |
| **단순성/관계** | E1 | "어떻게 하면 복잡한 것 없이 안전하게 가족을 챙길까?" |
| **정확성/데이터** | E2 | "어떻게 하면 잘못된 정보 없이 정확하게 결정할까?" |
| **소속감/관성** | N1 | "어떻게 하면 지금의 안정적인 상태를 유지할까?" |

## 본 프로젝트에서 의미

### 핵심 타깃 3 (Phase 1~2)

- **C1** (가성비 최적화) — 메인 사용자, 매출 핵심
- **C2** (건강 계기) — 진입 트래픽, 신뢰 검증
- **A2** (트렌드) — 바이럴 + SEO 자연 진입

### 확장 타깃 3 (Phase 3~4)

- **E1** (디지털 약자) — 음성 UI/단순 모드 도입 시
- **E2** (데이터 검증) — 데이터 신뢰 자산 강화 시
- **N1** (브랜드 충성) — 비교 데이터 권위 확보 후 전환

### 페르소나별 우선 기능

| 페르소나 | 최우선 기능 | 보조 기능 |
|---|---|---|
| C1 | F1 HFF-Hub Engine | F4 Data Trust |
| C2 | F2 Anti-BS Dashboard | F1 |
| A2 | F3 Viral Engine | F2 |
| E1 | F1 (단순 모드) | F4 |
| E2 | F4 Data Trust | F2 |
| N1 | F2 + F4 (전환 트리거) | F1 |

## 결론·시사점

- 6명을 동시에 만족시키는 단일 UI는 불가능 → **단일 엔진 + 페르소나별 진입 모드** 구조 필요.
- C1을 위한 정밀도, C2를 위한 신뢰도, A2를 위한 공유성은 서로 충돌하지 않으나 **첫 화면에서 동시 표현**해야 한다 → UX 설계의 가장 큰 도전.
- 각 페르소나 상세는 [`personas/`](../personas/) 디렉터리.
